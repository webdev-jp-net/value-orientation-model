# 価値志向モデル（Value Orientation Model）

## 概要
このモデルは、自分がどのような判断の引き受け方・関係の持ち方をすると心地良いかを可視化し、
選択的にそうしている自分を把握することを目的とした志向プロファイルです。

自己理解や心理的リソースを運用の補助ツールとして利用できます。
他者の志向も可視化し相関図にすると、集団の志向や棲み分けを構造的に捉える参考資料にもなります。

なお、このモデルは評価の正しさ・能力・成果の大小を測るものではありません。

## 構造
このモデルは、価値判断における志向を**縦軸・横軸からなる2次元構造**として定義する。

- 縦軸：帰属志向（valueLocus）
- 横軸：関係性志向（boundary）

UIへの軸ラベル表記は、意味の正しさの優先度を下げ
心理学に馴染みのないユーザーが直感で想起しやすいことや優劣と紐づかないことの優先度を重視している。
そのため、正しい概念は仕様書で補完している。

### 帰属志向
判断・決定・結果に対する責任の帰属をあらわした志向です。
オーナーシップ ←→ コンセンサス

#### 軸の定義

##### オーナーシップ
オーナーシップ志向は、責任を自分が主体的に引き受ける選好です。
本来の「オーナーシップ」どおりの意味で、「リーダーシップ」とは別の概念です。
このモデルにおけるリーダーシップは関係性軸の融合と帰属軸のオーナーシップが両方とも高く融合コストを他者に置いている状態です。

端点に近づくと顕在化する傾向：
- 責任関与が深く主体的ですが、対ストレスリソースは高燃費で消耗しやすい傾向があります。
- 決定責任のみに限定した概念ではありません。内在化した価値を創出すること、決定責任を持つこと、いずれもオーナーシップ志向と連結しています。
- 責任関与が深い延長で設計や過程にもタッチする場面があるため、過程も価値対象としてキャッチアップしやすい傾向があります。
  - 端点へ極端に近いと、こだわりすぎて効率化を犠牲にする場合があります。
- 0→1の価値創出と親和性が高い特性です。一方で、自発的要素のないタスクへモチベーションを見い出しにくい傾向があります。

##### コンセンサス
コンセンサス志向は、総和をつくり責任を場や合意へ移譲する選好です。

端点に近づくと顕在化する傾向：
- 責任関与が浅く依存的ですが、対ストレスリソースは低燃費で高頻度・広範囲へ関与しやすい特性でもあります。
- 詳細把握や品質管理は専門家に委ね、接続や連携に重心を置く行動傾向があります。
- 合意形成はすりあわせ対象があることで成り立つため、結論やアウトプットなど外在化した成果にアンテナ感度が高い傾向があります。
  - 端点へ極端に近いと、過程に関与する概念そのものを忘れてしまう場合もあります。
- 0→1の価値創出は苦手ですが、補助的関与にもモチベーションの波を持ち込まず対処できる傾向があります。

#### ポイント算出方法
各軸に関連する変数の差分を、プロット座標とする。

- **帰属志向（Y軸）**: `オーナーシップ` - `コンセンサス`
- **関係性志向（X軸）**: `融合` - `自立`

それぞれの変数は、5〜10問の設問回答の合算（各 -10〜10程度）として算出される。
座標レンジ：オーナーシップ（20）←→ コンセンサス（-20）、融合（20）←→ 自立（-20）

### 関係性志向
自分以外のモノ・人・現象との関係性をあらわした志向です。
自立 ←→ 融合

#### 軸の定義

##### 自立
自立は、ダイバーシティを選好する志向です。
心理学本来の「自立」とは厳密に一致していません。融合の対角へダイバーシティを配置したときの誤認バイアスを軽減する目的で「自立」を採用しています。

端点に近づくと顕在化する傾向：
- 概念や個体の特性が独立性を認知できる構造を望み、混同や同質化をストレスに感じます。
  - 人間関係に限定せず、人・モノ・コトといった概念の混同や同質化にもストレスを感じます。
  - 肯定・否定・観測（無評価）の基準は内在化しており外から推測しにくいため、わからない人とラベリングされやすい。
- 同質化と社交性は異義です。したがって単独選好と自立志向も異義です。

##### 融合
融合は、一体感を選好する志向です。

端点に近づくと顕在化する傾向：
- 状態や嗜好が同質に近づく一体感を望み、拒否や異質にストレスを感じます。
  - 状態を収束させるための変化を望み調整へ働きかけます、ストレス時は異質を遠ざけたりもします。
  - 収束のための変化は、自分で対応する場合と外部へ求める場合の両方があります。

#### ポイント算出方法
（上記「帰属志向」のポイント算出方法を参照）
自立（-20）←→ 融合（20）

## パーソナルプロットの入力仕様

このモデルのマトリクスへプロットするドットのことを、「パーソナルプロット」と呼びます。

`名前`, `オーナーシップ`, `コンセンサス`, `自立`, `融合` の5項目を1セットのレコード（パーソナルプロットの構成要素）として扱います。

### ユーザー向けUX

このモデルのパーソナルプロットは、文章問題への5段階フィーリング回答の累積で作成します。  
帰属志向、関係性志向の2つの軸に対して、それぞれ10問の文章問題を設けます。

文章問題ではシチュエーションの提示があり、「否定」から「肯定」へのフィーリングを5段階の段階選択からの単一選択です。  
設問数は、最小限に絞りません。これは、回答者が曖昧に決めてよいと直感できるUXを成立させる意図です。  
選択肢には中立点を含めます。これは、中立的である本質を正しく値へ反映する意図で設けます。

### インポートUI

モデルの挙動確認や他者のプロット再現を目的として、以下の直接入力インターフェイスを設置します。

- **カンマ区切りによる一括入力**:
  `オーナーシップ`, `コンセンサス`, `自立`, `融合` の4つの数値（各 -14〜14）は、カンマ区切りによる一括テキスト入力（例: `10, 5, -5, 10`）により行います。

## 注意事項
- 各観点は独立しており、相互に補正・優劣付けを行わない
- 行動結果や能力評価を直接示すものではない
- 可視化手法（例：マトリクス表示）は表現手段であり、モデルの本質ではない

## 用途例
- 価値観の違いによる判断摩擦の整理
- チーム内の意思決定スタイルの理解
- 個人・組織の判断傾向の可視化
