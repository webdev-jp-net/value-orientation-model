# 価値志向モデル（Value Orientation Model）

## 概要
このモデルは、自分がどのような判断の引き受け方・関係の持ち方をすると心地良いかを可視化し、
選択的にそうしている自分を把握することを目的とした志向プロファイルです。

自己理解や心理的リソースを運用の補助ツールとして利用できます。
他者の志向も可視化し相関図にすると、集団の志向や棲み分けを構造的に捉える参考資料にもなります。

なお、このモデルは評価の正しさ・能力・成果の大小を測るものではありません。

## 構造
このモデルは、価値判断における志向を**縦軸・横軸からなる2次元構造**として定義する。

- 縦軸：帰属志向
- 横軸：関係性志向

UIへの軸ラベル表記は、意味の正しさの優先度を下げ
心理学に馴染みのないユーザーが直感で想起しやすいことや優劣と紐づかないことの優先度を重視している。
そのため、正しい概念は仕様書で補完している。


### 帰属
責任や決定をどこに置くか
オーナーシップ ←→ コンセンサス

オーナーシップ(100) = 自分の責任として引き取る
コンセンサス(0) = 場の総和をつくる


#### 構造論理
状態や特性、不確実性を自ら把握し、価値の成立条件を内在化して捉えようとする志向。
(0 - 50 この志向が強いほど50に近づく)

#### プロセス
結論と過程の両方へ注目する志向。
(0 - 50 この志向が強いほど50に近づく)

### 関係性
他者や場との境界をどう扱うか
自立 ←→ 融合

自立(0) = 特性保存を尊重する
融合(100) = 一体感をつくる

#### 人物
人物の属性や信頼感を取り入れた価値判断を行う志向。
(0 - 50 この志向が強いほど50に近づく)

#### 社会的調和
ターゲットの嗜好を選択的に優先し、そこへ融合させる調整志向。トレンド選好。
(0 - 50 この志向が強いほど50に近づく)

## 注意事項
- 各観点は独立しており、相互に補正・優劣付けを行わない
- 行動結果や能力評価を直接示すものではない
- 可視化手法（例：マトリクス表示）は表現手段であり、モデルの本質ではない


## 用途例
- 価値観の違いによる判断摩擦の整理
- チーム内の意思決定スタイルの理解
- 個人・組織の判断傾向の言語化
