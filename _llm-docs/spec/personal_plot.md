# パーソナルプロット定義と入力仕様 (PersonalPlot Specification)

マトリクス上にプロットされるドットの最小単位である「パーソナルプロット（PersonalPlot）」のデータ実態および、その入力に関する技術仕様。

## データ構成 (Entity Definition)

1つのパーソナルプロットは、以下の5つの項目を1セットのレコードとして保持する。

| 項目名         | 英語名称（コード） | 説明               |
| -------------- | ------------------ | ------------------ |
| 表示名         | `displayName`      | ラベル表示名       |
| オーナーシップ | `ownership`        | Y軸の正方向スコア |
| コンセンサス   | `consensus`        | Y軸の負方向スコア |
| 自立           | `diversity`        | X軸の負方向スコア |
| 融合           | `identityFusion`   | X軸の正方向スコア |

※各スコアの算出根拠は [model_definition.md](./model_definition.md) を参照。

## ユーザー入力 UX (Survey / Editor)

パーソナルプロットの作成・更新には、以下のインターフェイスを提供する。

### 設問回答による作成 (Survey)
- 20問の文章問題に対する5段階のフィーリング回答（-2 〜 +2）の累積によってスコアを算出する。
  - 各設問ではシチュエーションの提示があり、それに対するフィーリングを測定する。
  - 設問数は、最小限に絞らない。これは、回答者が曖昧に決めてよいと直感できるUXを成立させる意図である。
- 選択肢には「どちらでもない」という中立点を含める。これは、中立的である本質を正しく値へ反映する意図で設ける。
- 設問の意図や詳細コンセプトについては `_llm-docs/question_concepts.md` を参照。

### 直接入力およびインポート (Import UI)
- モデルの再現や挙動確認を目的として、数値を直接入力する手段を提供する。
- **CSV形式**: `ownership, consensus, diversity, identityFusion` の順で4つの数値をカンマ区切りで入力する。

## バリデーション (Validation)

- **必須項目**: `displayName` および4つの全数値。
- **保存条件**: グループ名が入力されており、かつ1名以上の「完全なデータ」を持つパーソナルプロットが存在すること。
